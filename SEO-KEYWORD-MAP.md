### 독립 자산 내부 링크 규칙

1. 홈페이지와 `/blog`는 질문 맥락에 맞는 연구·도구·자료 허브를 연결하고, 모든 자산을 같은 문단에 나열하지 않는다.
2. `피아노 코드` 정보 글은 코드표로, 비용 비교 글은 레슨비 계산기와 가격 페이지로 연결한다.
3. 초보·교재·진도 글은 수준별 로드맵으로, 연습 습관 글은 연습 플래너로 연결한다.
4. 검색수요를 인용하는 페이지는 보고서 URL을 연결하고 수치의 관측기간과 추정치 성격을 함께 밝힌다.
5. 연구 보고서는 방법론·수정 이력·편집 정책을 연결하고, 다운로드 파일만 고립된 진입점으로 만들지 않는다.
6. 도구·자료 페이지는 관련 수업 또는 가격 페이지를 설명형 링크로 연결하되 계산 결과를 상담·성과 보장으로 표현하지 않는다.

## 카니벌라이제이션 방지 규칙

1. 홈페이지는 `피아노 레슨`, 개인 페이지는 `피아노 개인 레슨`만 대표 키워드로 사용한다.
2. 상세 가격표와 포함 항목은 `/pricing`이 소유한다. 다른 페이지는 가격 요약과 가격 페이지 링크만 제공한다.
3. `성인 피아노 학원`은 실제 학원 서비스명이 아니다. `/blog/lesson-guide/adult-piano-tutoring`의 비교 의도로만 사용한다.
4. `피아노 학원`도 학원으로 자칭하지 않고 학원과 개인 레슨을 비교하는 정보 글에서만 사용한다.
5. `피아노 악보`는 실제 악보 파일·저작권·곡별 답변이 있을 때만 독립 페이지를 만든다. 곡 추천 글에 검색량만 보고 억지로 배정하지 않는다.
6. 지역 페이지는 실제 방문 가능 범위만 다룬다. 서비스하지 않는 지역명을 넣어 문을 늘리지 않는다.

## 페이지 작성 기준

모든 상업·정보 페이지는 다음 기준을 충족해야 한다.

- 대표 키워드: 고유한 title, H1, 첫 설명 문단, meta description에 포함
- 보조 키워드: 실제로 답하는 H2·본문·FAQ·내부링크 문구에 자연스럽게 포함
- 첫 문장: 해당 수업이나 질문에 대한 직접 답변으로 시작
- 본문: 검색 의도에 필요한 대상, 방식, 비용 또는 준비 기준을 구체적으로 설명
- FAQ: 화면에 보이는 질문·답변과 FAQ 구조화 데이터가 일치
- 내부링크: 정보 글에서 관련 수업 페이지, 수업 페이지에서 가격·선생님·관련 정보 글로 연결
- 금지: 같은 문구 반복, 서비스와 다른 `학원` 자칭, 확인되지 않은 지역·성과·합격 보장, 제공하지 않는 악보 키워드 사용

## 페이지 권위(PA) 집중 규칙

PA는 Google이 공개한 단일 점수가 아니라 외부 SEO 도구가 링크 그래프 등을 바탕으로 추정하는 페이지 단위 지표다. 이 문서에서는 점수를 임의로 만들지 않고, Google이 확인할 수 있는 페이지별 주제 소유권과 크롤 가능한 내부 링크를 강화한다.

1. `/lessons` 통합 페이지는 만들지 않는다. 핵심 키워드 `피아노 레슨`은 홈페이지가 계속 소유하고, 여섯 개 상업 랜딩은 서로 다른 확장 키워드만 담당한다.
2. 모든 상업 랜딩은 화면에 작성·검토자, 최종 검토일, 직접 답변, 담당 범위와 한계를 공개한다.
3. 상업 랜딩의 `WebPage`와 `Service`, 관련 정보 글의 `BlogPosting`을 JSON-LD `@id`로 연결한다. 구조화 데이터는 화면에 보이는 관계만 표현한다.
4. 32개 공개 글은 모두 최소 한 개의 관련 상업 랜딩으로 연결한다. 비용 비교와 직접 관련된 `choosing-piano-tutor`, `academy-vs-tutoring`, `tutoring-time-guide`, `adult-piano-tutoring`, `elementary-piano-tutoring`은 `/pricing`도 함께 연결한다.
5. `/blog`와 여섯 카테고리 허브는 작성자, 최신 검토일, 독자, 담당 범위와 제외 범위를 화면과 `CollectionPage`에 반영한다.
6. 링크 문구는 `자세히 보기`만 단독으로 쓰지 않고 `피아노 레슨비 확인하기`처럼 대상 페이지의 내용을 설명한다.
7. 외부 링크와 실제 검색 성과는 코드로 만들 수 없다. 배포 뒤 Search Console의 URL별 노출·클릭·색인 상태와 외부 링크를 기준으로 평가하며, Moz 등 제3자 PA는 보조 추세만 확인한다.

## 권위 자산 성과 판정

독립 페이지와 데이터 파일을 배포한 사실만으로 DA, PA 또는 Google 순위가 오르지는
않는다. 기술 검증과 검색 성과를 분리해 판정한다.

| 판정 영역     | 확인 항목                                                                |
| ------------- | ------------------------------------------------------------------------ |
| 배포 무결성   | 200 응답, canonical, H1, 구조화 데이터, sitemap.xml·llms.txt 포함        |
| 데이터 재현성 | 원본 스냅샷 SHA-256, CSV 행·필드·바이트·해시, 데이터 사전과 수정 이력    |
| 이용 가치     | 코드 조회·비용 환산·연습 계획 입력, PNG·CSV·인쇄/PDF 동작                |
| 검색 성과     | Search Console의 URL별 색인, 노출, 클릭, 검색어와 실제 자연 유입         |
| 외부 권위     | 관련 기관·교육자·매체의 자연 링크와 인용, 링크 출처의 주제 적합성        |
| 제3자 지표    | Moz DA/PA 등 동일 도구·동일 기간의 보조 추세이며 목표 달성의 보증값 아님 |
